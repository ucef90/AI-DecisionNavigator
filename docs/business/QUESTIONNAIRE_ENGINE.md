# Questionnaire Engine

## Objectif

Construire un moteur dynamique de questionnement permettant :
- d’analyser les besoins métier ;
- de qualifier les projets ;
- de détecter les risques ;
- de structurer la réflexion ;
- de guider la décision.

---

# Structure du questionnaire

## Bloc 1 — Besoin métier

Questions :
- Quel problème cherche-t-on à résoudre ?
- Quels irritants existent ?
- Quel résultat est attendu ?
- Qui est concerné ?

---

## Bloc 2 — Workflow

Questions :
- Comment fonctionne le processus actuel ?
- Où sont les tâches manuelles ?
- Où sont les lenteurs ?
- Où sont les validations ?

---

## Bloc 3 — Qualification IA

Questions :
- Les règles sont-elles connues ?
- Le besoin nécessite-t-il du NLP ?
- Le besoin nécessite-t-il de classifier ?
- Le besoin nécessite-t-il de générer du contenu ?
- Le besoin nécessite-t-il plusieurs actions ?

---

## Bloc 4 — Données

Questions :
- Les données existent-elles ?
- Où se trouvent-elles ?
- Sont-elles accessibles ?
- Sont-elles sensibles ?

---

## Bloc 5 — Risques

Questions :
- Quel est le risque si l’IA se trompe ?
- Existe-t-il un risque RGPD ?
- Une validation humaine est-elle nécessaire ?

---

# Vision future

Le moteur devra :
- adapter les questions ;
- détecter les incohérences ;
- générer des recommandations ;
- proposer des workflows ;
- produire des synthèses automatiques.
